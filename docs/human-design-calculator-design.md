## Problem Statement

The bttb app has a beautiful 8-chapter Human Design educational section with interactive SVG visualizations (bodygraph, type cards, profile matrix, timelines), but zero personalization. Users learn about HD in general but cannot calculate or explore their own chart. They must leave the app for English-only tools like mybodygraph.com. The Vietnamese HD community has no native-language interactive chart calculator, and no existing tool in any language combines calculation + interactive exploration + AI-powered interpretation.

## What Makes This Cool

Your personal bodygraph IS the navigation. You enter your birth data, and the chart materializes with your specific activations. Click any center, channel, or gate to understand what it means FOR YOU. Personality activations (black/conscious) vs Design activations (red/unconscious) show the duality of your nature. Today's planetary transits pulse as overlays on your natal chart, showing which gates are being activated RIGHT NOW. The copy prompt + Claude skill generates a personalized Vietnamese reading that no other tool offers. The chart becomes a daily practice tool, not a one-time result.

## Constraints

- Must use `astronomy-engine` (already in project) for planetary calculations, no external API dependencies
- Must support Vietnamese + English bilingual content throughout
- Birth time accuracy is critical (HD charts shift with minutes). Must clearly communicate this to users.
- Birth location required for UTC conversion. Need timezone lookup from coordinates.
- Must follow existing patterns: API routes like `/api/bazi/calculate`, components like `BodygraphSvg`, Prisma models like `BaziReading`
- The 64-gate wheel mapping is a fixed lookup table (each gate = 5.625 degrees of ecliptic)
- 13 celestial bodies: Sun, Earth, Moon, North Node, South Node, Mercury, Venus, Mars, Jupiter, Saturn, Uranus, Neptune, Pluto
- Two calculation timestamps: Personality (birth time) and Design (~88 solar degrees before birth)

## Premises

1. The calculation engine uses `astronomy-engine` for planetary positions + a fixed gate lookup table. 13 bodies at 2 timestamps gives gate activations, from which we derive type, authority, profile, defined centers, and channels.
2. The interactive chart reuses the existing `BodygraphSvg` component with real calculated data. Clicking centers/channels/gates shows YOUR specific activations with Vietnamese descriptions.
3. The Claude skill + copy prompt is the killer feature. Structured text summary + HD analysis skill = personalized Vietnamese reading.
4. Birth location required for UTC conversion. Time of birth is critical.

## Cross-Model Perspective

The Claude subagent's cold read surfaced a key insight: the transit overlay transforms the chart from a one-time calculation into a daily practice tool. "Interpret my chart in the context of today's transits" is a question people ask repeatedly, not once. This drives daily retention.

The subagent also identified "navigation" as the signal word in the user's answers, confirming the chart-as-UI architectural decision. Estimated the calculation bridge at ~400-500 lines of TypeScript, with the derivation logic (gates -> channels -> centers -> type/authority/profile) as the real precision work.

## Approaches Considered

### Approach A: Minimal Viable Chart (S effort)
Calculation engine + API route + results page. Reuses BodygraphSvg with real data. Copy prompt button. No interactivity.
- Pros: Ships fastest, proves calculation works
- Cons: Not interactive, no Vietnamese per-gate descriptions
- Completeness: 4/10

### Approach B: Interactive Personal Bodygraph (M effort)
Full calculation + interactive chart with click-to-explore. Vietnamese descriptions. Personality vs Design color distinction. Copy prompt + Claude skill.
- Pros: Delivers core vision, differentiator over existing tools
- Cons: More complex, needs gate description content
- Completeness: 7/10

### Approach C: Living Bodygraph with Transits (L effort) -- CHOSEN
Everything in B + real-time transit overlay, daily energy context, cross-reference with Bazi. Daily practice tool.
- Pros: 10x version, daily retention, unique in any language
- Cons: More scope, transit interpretation content needed
- Completeness: 9/10

## Recommended Approach

**Approach C: Living Bodygraph with Transits** -- built incrementally in 3 phases:

### Phase 1: Calculation Engine + API (foundation)

**New files:**

1. `src/lib/human-design/gate-table.ts` (~80 lines)
   - Fixed lookup: 64 gates mapped to ecliptic degree ranges
   - Gate wheel order starting at Gate 41 (2 degrees 00' Aquarius = 332.0 degrees ecliptic)
   - Each gate spans 5.625 degrees, each line spans 0.9375 degrees
   - Function: `longitudeToGate(longitude: number) => { gate: number, line: number }`

2. `src/lib/human-design/calculate-planets.ts` (~150-180 lines)
   - Uses `astronomy-engine` to get ecliptic longitude for 13 bodies
   - Two timestamps: Personality (birth UTC) and Design (88 solar degrees before birth)
   - Design timestamp: calculate birth Sun longitude, subtract 88 degrees (with 360 wrap, e.g., birth Sun at 30 degrees -> target 302 degrees). Use `astronomy-engine`'s `SearchSunLongitude(targetLongitude, startDate)` where startDate = 100 days before birth. The function searches FORWARD from startDate, finding the moment the Sun reaches the target longitude. This should converge to a date ~88 days before birth. If search fails to return a date, return error: "Could not determine Design date."
   - Function: `calculatePlanetaryPositions(birthDateUTC: Date) => PersonalityDesignActivations`

3. `src/lib/human-design/derive-chart.ts` (~200 lines)
   - Takes 26 gate activations (13 bodies x 2 timestamps)
   - Determines defined channels (both gates of a channel activated by any planet)
   - Determines defined centers (connected to at least one defined channel)
   - Derives Type (motor centers = Sacral, Heart, Solar Plexus, Root):
     - Generator: Sacral defined, no motor-to-Throat connection
     - Manifesting Generator: Sacral defined AND motor-to-Throat connection (via any path through defined centers)
     - Manifestor: Sacral undefined, motor-to-Throat connection (Heart, Solar Plexus, or Root reaching Throat through defined channels -- Sacral excluded because it's already ruled undefined)
     - Projector: Sacral undefined, no motor-to-Throat, at least one center defined
     - Reflector: No centers defined
   - Note: Motor-to-Throat must trace through defined channels/centers, not just direct connections
   - Derives Authority hierarchy: Solar Plexus > Sacral > Spleen > Heart/Ego > G/Self > Throat > None (Lunar for Reflectors)
   - Derives Profile: Line of Personality Sun + Line of Design Sun (e.g., 4/6)
   - Derives Incarnation Cross: Personality Sun gate, Personality Earth gate, Design Sun gate, Design Earth gate
   - Function: `deriveChart(activations: PersonalityDesignActivations) => HumanDesignChart`

4. `src/lib/human-design/channel-table.ts` (~60 lines)
   - Maps 36 channels to their gate pairs and connected centers
   - References existing `HD_CHANNELS` from `human-design-data.ts` but adds gate-pair lookup
   - Function: `getChannelByGates(gateA: number, gateB: number) => Channel | null`

5. `src/lib/human-design/index.ts` (~30 lines)
   - Orchestrator: `computeHumanDesign(birthInput: HDBirthInput) => HumanDesignChart`
   - HDBirthInput: { year, month, day, hour, minute, timezone: string (IANA format, e.g. "Asia/Ho_Chi_Minh"), latitude, longitude }
   - Note: IANA timezone is authoritative for UTC conversion. Latitude/longitude are used for timezone suggestion in the UI and stored for reference. If user overrides the auto-detected timezone, the override takes precedence.

6. `src/app/api/human-design/calculate/route.ts` (~60 lines)
   - POST endpoint following Bazi pattern
   - Input validation: year (1900-2100), month (1-12), day (1-31), hour (0-23), minute (0-59), timezone (valid IANA string), latitude (-90 to 90), longitude (-180 to 180)
   - Unknown birth time handling: if hour/minute omitted, default to 12:00 noon local time and flag result with `birthTimeUnknown: true` (chart page shows warning: "Birth time unknown -- chart may be inaccurate, especially for Moon and Ascendant gates")
   - Converts local time to UTC using IANA timezone, calls computeHumanDesign
   - Returns full chart JSON

7. `prisma/schema.prisma` additions:
   ```
   // Add to existing User model:
   // hdReadings HumanDesignReading[]

   model HumanDesignReading {
     id        String   @id @default(cuid())
     userId    String?
     user      User?    @relation(fields: [userId], references: [id])
     name      String
     gender    String?
     birthDate DateTime
     birthTime String
     birthTimeUnknown Boolean @default(false)
     birthPlace String
     latitude  Float
     longitude Float
     timezone  String
     result    String   // JSON string, matching BaziReading convention
     createdAt DateTime @default(now())
     updatedAt DateTime @updatedAt
   }
   ```

**New/modified files:**

   - Color coding: Personality activations = black/dark, Design activations = red/warm
   - Both Personality + Design on same gate = striped/gradient
   - Vietnamese + English toggle

3. Enhanced `src/components/human-design/BodygraphSvg.tsx`
   - All enhancements are additive via optional props to maintain backward compatibility with existing educational pages
   - New optional props: `activations?: PersonalityDesignActivations`, `onGateClick?: (gateId: number) => void`, `transitActivations?: GateActivation[]`, `showGateLabels?: boolean`
   - When `activations` is provided: show Personality (black) vs Design (red) color distinction
   - When `activations` is omitted: existing behavior unchanged (defined/undefined only)
   - Add gate number labels on channels (when `showGateLabels` is true)
   - Add hover states showing gate name + line
   - Add click handlers for gates (not just centers)
   - Add animation: gates pulse softly when first rendered

4. `src/components/human-design/ChartSummaryPanel.tsx` -- Left panel
   - Type badge with strategy + signature
   - Authority description
   - Profile archetype (e.g., "4/6 - Cơ Hội / Hình Mẫu")
   - Incarnation Cross name
   - Defined vs undefined centers list
   - Active channels list

5. `src/components/human-design/GateDetailPanel.tsx` -- Right panel
   - Gate number, name (Vietnamese + English), I Ching hexagram reference
   - Which planet activates it, Personality or Design
   - Line description
   - Channel connection (if applicable)
   - Center it belongs to
   - Vietnamese interpretation text

6. `src/components/human-design/HDPromptGenerator.tsx` -- Copy prompt
   - Generates structured text summary of the full chart
   - Includes: type, authority, profile, incarnation cross, all gate activations with planets, defined centers, defined channels
   - Copy-to-clipboard button
   - Format designed for Claude skill consumption

7. Gate description data: `src/lib/human-design/gate-descriptions.ts`
   - **Minimum viable content for Phase 2 launch:** 64 gates with Vietnamese + English names and 1-2 sentence theme descriptions. Seed from existing I Ching hexagram data in the DB (gate numbers correspond to hexagram numbers).
   - Shadow/gift/siddhi keywords per gate (64 entries)
   - Line descriptions (384 entries) are Phase 2.5 content -- not required for launch
   - When line description is unavailable, `GateDetailPanel` shows gate-level description with note: "Chi tiet dong dang duoc cap nhat" (Line details coming soon)
   - Cross-references to I Ching hexagram numbers (already in the codebase)
   - **Content strategy:** Gate themes seeded from I Ching hexagram DB, then refined with HD-specific language. Line descriptions added incrementally, prioritizing gates that appear most frequently in charts (Sun/Earth gates of common incarnation crosses).

### Phase 3: Living Chart with Transits (daily practice)

1. `src/lib/human-design/calculate-transits.ts`
   - Calculate current planetary positions (same 13 bodies)
   - Map to gates using same lookup table
   - Return which gates/channels/centers are transit-activated right now

2. `src/components/human-design/TransitOverlay.tsx`
   - Semi-transparent pulsing overlay on the bodygraph
   - Transit activations shown as glowing/animated gates
   - "Today's Energy" summary panel
   - Shows which of YOUR undefined centers are being activated by transits

3. `src/app/human-design/chart/[id]/transits/page.tsx` (or toggle on chart page)
   - Date picker to view transits for any day
   - "Today" button for current transits
   - Vietnamese description of transit meaning in context of natal chart

4. Cross-system links (lightweight, not a unified view):
   - Link to Bazi day energy (same birth date, different system's perspective)
   - Link to I Ching daily casting in context of HD transits
   - Note: A unified "Today's Energy" view combining all three systems is a separate future design. Phase 3 only adds navigation links between the three systems, not a merged interface.

## Open Questions

1. **Timezone handling:** Recommend IANA timezone strings (e.g., "Asia/Ho_Chi_Minh") since UTC offsets don't handle DST correctly for historical dates. Use `tzlookup` or similar to auto-detect from lat/lng, with manual override available. ~RESOLVED in design: use IANA format.
2. **Gate descriptions in Vietnamese:** ~RESOLVED: Seed 64 gate themes from I Ching hexagram DB data. Expand incrementally. 384 line descriptions are Phase 2.5 content.
3. **88-degree Design calculation precision:** ~RESOLVED: Use `SearchSunLongitude` with 100-day backward limit and 360-degree wrap handling.
4. **Location search:** Full geocoding API (Google/OpenStreetMap) or a simpler approach (major Vietnamese cities + manual lat/lng input)? Recommend: start with a curated list of ~50 Vietnamese cities + manual lat/lng/timezone input for other locations. Add geocoding API later.
5. **Incarnation Cross names:** ~RESOLVED for Phase 1-2: Derive cross from gate numbers only (e.g., "Right Angle Cross of the Sphinx: 1-2 | 7-13"). Full 768-name lookup table is a separate content task for Phase 2.5+.

## Success Criteria

- Calculate a chart that matches mybodygraph.com for the same birth data (same type, profile, defined centers, channels)
- Interactive bodygraph where clicking any element shows personalized information
- Vietnamese descriptions for all 5 types, 7 authorities, 12 profiles, 9 centers, 36 channels
- Copy prompt generates text that Claude + HD skill can interpret into a meaningful reading
- Transit overlay shows current planetary activations on the natal chart
- Page loads in under 2 seconds. Calculation target: under 1 second on warm server. Note: 26 astronomical calculations may be slower on serverless cold start. Fallback: if server-side exceeds 2 seconds, consider client-side calculation (astronomy-engine runs in browser).
